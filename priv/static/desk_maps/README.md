# Desk map base image

`office.png` in this folder is the static background `Manfrod.DeskMap`
composites the daily occupant name tags onto (see
`Manfrod.DeskMap.render_png/1`). It's not hand-drawn — it's generated by
`priv/repo/generate_desk_map.exs`, which also sets each bookable desk's
`map_x`/`map_y` in the database to match. Don't edit the PNG directly; edit
the script and re-run it.

## How to add a new desk to the map

1. Make sure the desk exists in the database first (via the `add_desk` tool,
   or add it to `priv/repo/seeds.exs` if it's part of the base layout).
2. Open `priv/repo/generate_desk_map.exs` and add an entry to the `desks`
   list: `{label, top_left_x, top_left_y, box_text_lines, permanent?}`.
   - `label` must match the desk's label in the database exactly.
   - `box_text_lines` is what gets drawn *inside* the box (usually the
     label + equipment, e.g. `["T7", "usb-c"]`) — this is baked into the
     image as static text, separate from the desk's `equipment` field in
     the database (which is what `list_desks`/`list_desk_reservations`
     read for their text answers). Keep the two in sync by hand when you
     change one.
   - `permanent?` (`true`/`false`): `true` bakes a fixed owner name into the
     box (like Agata/Franek) and skips it entirely below — permanently
     assigned desks never get a dynamic overlay, there's nothing to render
     on top.
3. If it's a normal bookable desk (`permanent? == false`), also add an entry
   to `bookable_positions` — the pixel spot (usually near the bottom of the
   box) where the daily occupant's name tag gets composited at render time.
4. Re-run the generator (see below).

## Changing an existing desk's box (move it, resize the layout, change the
baked-in text) or removing one from the map

Same file — edit or delete its entry in `desks` (and `bookable_positions` if
applicable), then re-run the generator. Removing an entry only affects the
*map picture*; it doesn't deactivate the desk for booking — use
`deactivate_desk` for that separately.

## Running the generator

```
mix run priv/repo/generate_desk_map.exs
```

It needs a live DB connection and boots the full app, so if `mix phx.server`
is already running locally, point it at a throwaway port to avoid a port
conflict:

```
PORT=45999 mix run priv/repo/generate_desk_map.exs
```

This overwrites `office.png` and updates `map_x`/`map_y` for every desk
listed in `bookable_positions` (idempotent — safe to re-run any time the
layout changes, not just once).

## Production

`office.png` ships inside `priv/static` as part of the release. Regenerating
it means running the script wherever it has DB access — either on the
production DB before the next deploy, or directly against the running
release's DB if `priv/static/desk_maps` is on a persistent volume — then
redeploying if it isn't.
